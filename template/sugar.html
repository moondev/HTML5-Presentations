About Sugar

A Quick Summary

Sugar is likely to be compared to Underscore.js as they both are support libraries for "functional" programming. The main differences are in their depth, syntax, and overall implementation. Sugar supports a much larger set of core utilities than that of Underscore.js. Those actively using Underscore.js will likely complain about the fact that Sugar extends Javascript objects. While I cannot argue this fact, Sugar does make an effort to maintain maximum compatibility as it will never overwrite methods that already exist. You can even manually override existing frameworks that have already extended native objects (Prototype, YUI, MooTools) with a simple call to Object.enableSugar().

Components of Sugar

  • Arrays
    • Cross-browser fixes for broken methods (indexOf, forEach, reduce...)
    • Enhanced (extended) support of methods (additional arguments)
    • Helper methods for manipulation
  • Strings
    • Cross-browser fixes for regex inconsistencies (split)
    • Multi-lingual support (unicode)
    • Helper methods for manipulation
  • Numbers
    • Shortcut methods to Math object methods
    • Number rounding to precision, number padding, number formatting
    • Intertwined with the date class for advanced date manipulations
  • Dates
    • Understanding of nearly all date strings
      • Text based, relative, absolute, timestamps
    • Date output in nearly any format
    • Helper methods for date manipulation and validation
  • Objects
    • Create hash-like objects (keys, values)
    • Type checking methods
  • Functions
    • Easily bind scope to functions
    • Delayed function calling
    • Throttled, non-blocking function closures
  • RegExp
    • Fixes inconsistencies in string splitting on regexes
    • Convenient escape method
    • Manipulation methods for generating RegExps

Components: String Methods

  • add(str, index = 0) Adds str at index. Negative values allowed.
  • at(index, loop = true) Gets the character(s) at a given index.
  • camelize(first = true) Converts underscores and hyphens to camel case. Capitalize first letter by default.
  • capitalize() Capitalizes the first character in the string.
  • chars(fn) Runs callback fn against each character in the string. Returns array of chars.
  • codes(fn) Runs callback fn against each character code in the string. Returns array of char codes. If given a string, converts to array of char codes.
  • compact() Compacts all white space in the string to a single space and trims ends.
  • dasherize() Converts underscores and camel casing to hyphens.
  • decodeBase64() Decodes the string from base64 encoding.
  • each(search, fn) Runs callback fn against each occurence of search. Returns array.
  • encodeBase64 Encodes the string into base64 encoding.
  • endsWith(find, caseSensitive = true) Returns true if the string ends with find
  • escapeRegExp() Escapes all RegExp tokens in the string.
  • escapeURL(param = false) Escapes characters in a string to make a valid URL. If param = true, will also escape valid URL characters for use as URL parameters.
  • first(n = 1) Returns the first n characters of the string.
  • from(index = 0) Returns a section of the string starting from index
  • hankaku(mode = 'all') Converts full-width characters (zenkaku) to half-width (hankaku). Mode accepts a (alphabet), n (numbers), k (katakana), s (spaces), p (punctuation), or all.
  • has(find) Returns true if the string matches find where find may be a RegExp or string.
  • hiragana(all = true) Converts katakana into hiragana. If all is false, only convert full-width katakana, otherwise convert all.
  • insert() Alias for add().
  • isBlank() Returns true if the string has a length of 0 or contains only whitespace.
  • katakana() Converts hiragana into katakana.
  • last(n = 1) Returns the last n characters of the string.
  • lines(fn) Runs callback fn against each line in the string. Returns an array of lines.
  • normalize() Returns the string with accented and non-standard Latin-based characters converted into standard letters.
  • pad(num = 0, padding = '') Pads both sides of a string where num is the number of characters on each side and padding is the character to pad with.
  • padLeft(num = 0, padding = '') Pads the left side of the string where num is the number of characters on each side and padding is the character to pad with.
  • padRight(num = 0, padding = '') Pads the right side of the string where num is the number of characters on each side and padding is the character to pad with.
  • removeTags(tag1, tag2, ...) Removes all HTML tags and their contents from the string. No arguments will remove all tags and nested content.
  • repeat(num = 0) Returns the string repeated num times.
  • reverse() Reverse the string.
  • split(separator, limit) Split the string by separator into an array. Native regex support fixed.
  • stripTags(tag12, tag2, ...) Strips all HTML tags from the string. Unlike removeTags, all child text remains intact.
  • titleize() Capitalizes all first letters.
  • to(index = end) Returns a section of the string ending at index.
  • toDate() Creates a date from the given string.
  • toNumber(base = 10) Convers the string into a number. Any value with a "." fill is converted to floating point, otherwise int.
  • toObject(sep1, sep2) Converts a string into an object. String could be formatted as URL GET parameters or piped and comma delimited.
  • trim() Removes leading and trailing whitespace from a string.
  • trimLeft() Removes leading whitespace from a string.
  • trimRight() Removes trailing whitespace from a string.
  • underscore() Converts hyphens and camel casing to underscores.
  • unescapeURL(partial = false) Restores escaped characters in a URL escaped string. If partial is true, will only unescape non-valid URL chars.
  • words(fn) Runs callback fn against each word in the string. Returns an array of words.
  • zenkaku(mode) Converts half-width characters (hankaku) to full-width (zenkaku). Mode accepts a (alphabet), n (numbers), k (katakana), s (spaces), p (punctuation), or all.

Example 1: Easily Parse Twitter JSONP Response

getLatestTweets(function(t) {
  var users = t.map('user').unique();
  var total = users.sum('statuses_count').format();
  var top   = users.max('followers_count').first();
  var count = top.followers_count.format();
  var since = Date.create(top.created_at);
  return 'Returned ' + users.length + ' Skookum team members with a total of ' + total + ' tweets.\n' +
         top.screen_name + ' is the most followed with ' + count + ' followers\n' +
         'and started tweeting ' + since.relative() + '.';
});
