About Sugar

A Quick Summary

Sugar is likely to be compared to Underscore.js as they both are support libraries for "functional" programming. The main differences are in their depth, syntax, and overall implementation. Sugar supports a much larger set of core utilities than that of Underscore.js. Those actively using Underscore.js will likely complain about the fact that Sugar extends Javascript objects. While I cannot argue this fact, Sugar does make an effort to maintain maximum compatibility as it will never overwrite methods that already exist. You can even manually override existing frameworks that have already extended native objects (Prototype, YUI, MooTools) with a simple call to Object.enableSugar().

Components of Sugar

  • Arrays
    • Cross-browser fixes for broken methods (indexOf, forEach, reduce...)
    • Enhanced (extended) support of methods (additional arguments)
    • Helper methods for manipulation
  • Strings
    • Cross-browser fixes for regex inconsistencies (split)
    • Multi-lingual support (unicode)
    • Helper methods for manipulation
  • Numbers
    • Shortcut methods to Math object methods
    • Number rounding to precision, number padding, number formatting
    • Intertwined with the date class for advanced date manipulations
  • Dates
    • Understanding of nearly all date strings
      • Text based, relative, absolute, timestamps
    • Date output in nearly any format
    • Helper methods for date manipulation and validation
  • Objects
    • Create hash-like objects (keys, values)
    • Type checking methods
  • Functions
    • Easily bind scope to functions
    • Delayed function calling
    • Throttled, non-blocking function closures
  • RegExp
    • Fixes inconsistencies in string splitting on regexes
    • Convenient escape method
    • Manipulation methods for generating RegExps

Components: String Methods

  • add(str, index = 0) Adds str at index. Negative values allowed.
  • at(index, loop = true) Gets the character(s) at a given index.
  • camelize(first = true) Converts underscores and hyphens to camel case. Capitalize first letter by default.
  • capitalize() Capitalizes the first character in the string.
  • chars(fn) Runs callback fn against each character in the string. Returns array of chars.
  • codes(fn) Runs callback fn against each character code in the string. Returns array of char codes. If given a string, converts to array of char codes.
  • compact() Compacts all white space in the string to a single space and trims ends.
  • dasherize() Converts underscores and camel casing to hyphens.
  • decodeBase64() Decodes the string from base64 encoding.
  • each(search, fn) Runs callback fn against each occurence of search. Returns array.
  • encodeBase64 Encodes the string into base64 encoding.
  • endsWith(find, caseSensitive = true) Returns true if the string ends with find
  • escapeRegExp() Escapes all RegExp tokens in the string.
  • escapeURL(param = false) Escapes characters in a string to make a valid URL. If param = true, will also escape valid URL characters for use as URL parameters.
  • first(n = 1) Returns the first n characters of the string.
  • from(index = 0) Returns a section of the string starting from index
  • hankaku(mode = 'all') Converts full-width characters (zenkaku) to half-width (hankaku). Mode accepts a (alphabet), n (numbers), k (katakana), s (spaces), p (punctuation), or all.
  • has(find) Returns true if the string matches find where find may be a RegExp or string.
  • hiragana(all = true) Converts katakana into hiragana. If all is false, only convert full-width katakana, otherwise convert all.
  • insert() Alias for add().
  • isBlank() Returns true if the string has a length of 0 or contains only whitespace.
  • katakana() Converts hiragana into katakana.
  • last(n = 1) Returns the last n characters of the string.
  • lines(fn) Runs callback fn against each line in the string. Returns an array of lines.
  • normalize() Returns the string with accented and non-standard Latin-based characters converted into standard letters.
  • pad(num = 0, padding = '') Pads both sides of a string where num is the number of characters on each side and padding is the character to pad with.
  • padLeft(num = 0, padding = '') Pads the left side of the string where num is the number of characters on each side and padding is the character to pad with.
  • padRight(num = 0, padding = '') Pads the right side of the string where num is the number of characters on each side and padding is the character to pad with.
  • removeTags(tag1, tag2, ...) Removes all HTML tags and their contents from the string. No arguments will remove all tags and nested content.
  • repeat(num = 0) Returns the string repeated num times.
  • reverse() Reverse the string.
  • split(separator, limit) Split the string by separator into an array. Native regex support fixed.
  • stripTags(tag12, tag2, ...) Strips all HTML tags from the string. Unlike removeTags, all child text remains intact.
  • titleize() Capitalizes all first letters.
  • to(index = end) Returns a section of the string ending at index.
  • toDate() Creates a date from the given string.
  • toNumber(base = 10) Convers the string into a number. Any value with a "." fill is converted to floating point, otherwise int.
  • toObject(sep1, sep2) Converts a string into an object. String could be formatted as URL GET parameters or piped and comma delimited.
  • trim() Removes leading and trailing whitespace from a string.
  • trimLeft() Removes leading whitespace from a string.
  • trimRight() Removes trailing whitespace from a string.
  • underscore() Converts hyphens and camel casing to underscores.
  • unescapeURL(partial = false) Restores escaped characters in a URL escaped string. If partial is true, will only unescape non-valid URL chars.
  • words(fn) Runs callback fn against each word in the string. Returns an array of words.
  • zenkaku(mode) Converts half-width characters (hankaku) to full-width (zenkaku). Mode accepts a (alphabet), n (numbers), k (katakana), s (spaces), p (punctuation), or all.

Components: Number Methods

  • Number.random(n1, n2) Returns a random integer between n1 and n2. If only 1 number is passed, the other will be 0. If none are passed, the number will be either 0 or 1.
  • abs() Returns the absolute value for the number.
  • ceil(precision = 0) Rounds the number up. Will round to the given precision.
  • chr() Returns a string at the code point of the number.
  • days() Takes an integer as days and returns milliseconds. Similar handling also applies to hours(), milliseconds(), minutes(), months, seconds(), weeks(), and years().
  • daysAfter(d) Returns a date n days after d, where n is the integer. d will accept a date object, timestamp, text format, dayAfter(), or dayFromNow(). d defaults to now. Similar handling also applies to hoursAfter(), millisecondsAfter(), minutesAfter(), monthsAfter, secondsAfter(), weeksAfter(), and yearsAfter().
  • daysAgo() Alias for daysBefore(). Similar handling also applies to hoursAgo(), millisecondsAgo(), minutesAgo(), monthsAgo, secondsAgo(), weeksAgo(), and yearsAgo(). These similar functions also have similar aliases of the form [timeframe]Before(), i.e. hoursBefore(), millisecondsBefore(), minutesBefore(), monthsBefore, secondsBefore(), weeksBefore(), and yearsBefore().
  • daysBefore(d) Returns a date n days after d, where n is the integer. d will accept a date object, timestamp, text format, dayAfter(), or dayFromNow(). d defaults to now. Similar handling also applies to hoursBefore(), millisecondsBefore(), minutesBefore(), monthsBefore, secondsBefore(), weeksBefore(), and yearsBefore().
  • daysFromNow(d) Alias for daysAfter(). Similar handling also applies to hoursFromNow(), millisecondsFromNow(), minutesFromNow(), monthsFromNow, secondsFromNow(), weeksFromNow(), and yearsFromNow(). These similar functions also have similar aliases of the form [timeframe]After(), i.e. hoursAfter(), millisecondsAfter(), minutesAfter(), monthsAfter, secondsAfter(), weeksAfter(), and yearsAfter().
  • downto(num, func) Returns an array containing numbers from the number down to num. Optionally calls fn callback for each number in that array.
  • floor(precision = 0) Rounds the number down. precision will round to the given precision.
  • format(comma, period) Formats the number to a readable string. comma is the character used for the thousands separator. period is the character used for the decimal point.
  • hex() Converts the number to hexidecimal.
  • isEven() Returns true if the number is even.
  • isMultipleOf(num) Returns true if the number is a multiple of num.
  • isOdd() Returns true if the number is odd.
  • ordinalize() Returns an ordinalized (English) string, i.e. "1st", "2nd", etc.
  • pad(place = 0, sign = false) Pads a number with "0" to place. sign allows you to force the sign as well (+05, etc).
  • pow(p = 1) Returns the number to the power of p.
  • round(precision = 0) Rounds a number to the precision of precision.
  • times(n) Calls fn a number of times equivalent to the number.
  • toNumber() Returns a number. This is mostly for compatibility reasons.
  • upto(num, fn) Returns an array containing numbers from the number up to num. Optionally calls fn callback for each number in that array.

Components: Array Methods

  • underscore() Converts hyphens and camel casing to underscores.
  • add(el, index) Adds el to the array. If index is specified, it will add at index, otherwise adds to the end of the array. Behaves like concat in that if el is an array it will be joined, not inserted. This method will change the array! Use include for a non-destructive alias.
  • all() Alias for every().
  • any() Alias for some().
  • at(index, loop = true) Gets the element(s) at the given index. When loop is true, overshooting the end of the array (or the beginning) will begin counting from the other end. Also accepts passing multiple indexes will get the elements at those indexes. Returns mixed.
  • average(map) Averages all values in the array. map may be a function mapping the value to be averaged or a string acting as a shortcut.
  • clone() Clones the array.
  • collect() Alias for map().
  • compact() Removes all instances of undefined, null, and NaN from the array. Empty strings and false are left untouched.
  • count(f) Counts all elements in the array that match f. f will match a string, number, array, object, or alternately test against a function or regex.
  • each() Alias for forEach().
  • eachFromFunction(index, fn, loop = false) Runs fn against elements in the array starting at index. Will run a full loop starting over at index = 0 if loop is true.
  • every(f, scope) Returns true if all elements in the array match f. scope is the this object. In addition to providing this method for browsers that don't support it natively, this enhanced method also directly accepts strings, numbers, deep objects, and arrays for f. all() provided as an alias.
  • exclude(f) Removes any element in the array that matches f. This is a non-destructive alias for remove(). It will not change the original array.
  • filter(f, scope) Returns any elements in the array that match f. scope is the this object. In addition to providing this method for browsers that don't support it natively, this enhanced method also directly accepts strings, numbers, deep objects, and arrays for f.
  • find(f, index = 0, scope = false) Returns the first element that matches f. f will match a string, number, array, object, or alternately test against a function or regex. Starts at index, and will continue from index = 0 if loop is true.
  • findAll(f, index = 0, scope = false) Returns all elements that match f. f will match a string, number, array, object, or alternately test against a function or regex. Starts at index, and will continue from index = 0 if loop is true.
  • first(num = 1) Returns the first element(s) in the array. When num is passed, returns the first num elements in the array.
  • flatten() Returns a flattened, one-dimensional copy of the array.
  • forEach(fn, scope) Iterates over the array, calling fn on each loop. scope becomes the this object. each() is provided as an alias. This method is only provided for those browsers that do not support it natively.
  • from(index) Returns a slice of the array from index.
  • groupBy(property) Groups the array by property. Will return an object with keys equal to the grouped values. property may be a mapping function, or a string acting as a shortcut.
  • has() Alias for some()
  • inGroups(num, padding) Groups the array into num arrays. padding specifies a value with which to pad the last array so that they are all equal length.
  • inGroupsOf(num, padding = null) Groups the array into arrays of num elements each. padding specifies a value with which to pad the last array so that they are all equal length.
  • include(el, index) Adds el to the array. This is a non-destructive alias for add(). It will not change the original array.
  • indexOf(f, offset) Searches the array and returns the first index where f occurs, or -1 if the element is not found. offset is the index from which to begin the search. In addition to providing this method for browsers that don't support it natively, this enhanced method can also perform deep finds on objects or arrays, and accept an iterating function on which to perform the search.
  • insert() Alias for add().
  • intersect(a) Returns an array containing the elements both arrays have in common.
  • isEmpty() Returns true if the array is empty. This is true if the array has a length of zero, or contains only undefined, null, or NaN.
  • last() Returns the last element(s) in the array. When num is passed, returns the last num elements in the array.
  • lastIndexOf(f, offset) Searches the array and returns the last index where f occurs, or -1 if the element is not found. offset is the index from which to begin the search. In addition to providing this method for browsers that don't support it natively, this enhanced method can also perform deep finds on objects or arrays, and accept an iterating function on which to perform the search.
  • least(map) Returns the elements in the array with the least commonly occuring value. map may be a function mapping the value to be checked or a string acting as a shortcut.
  • map(fn, scope) Maps the array to another array containing the values that are the result of calling fn on each element. scope is the this object. In addition to providing this method for browsers that don't support it natively, this enhanced method also directly accepts a string, which is a shortcut for a function that gets that property on each element. collect() is provided as an alias.
  • max(map) Returns the elements in the array with the greatest value. map may be a function mapping the value to be checked or a string acting as a shortcut.
  • min(map) Returns the elements in the array with the lowest value. map may be a function mapping the value to be checked or a string acting as a shortcut.
  • most(map) Returns the elements in the array with the most commonly occuring value. map may be a function mapping the value to be checked or a string acting as a shortcut.
  • none(f) Returns true if none of the elements in the array match f. f will match a string, number, array, object, or alternately test against a function or regex.
  • randomize() Randomizes the array.
  • reduce(fn, init) Reduces the array to a single result. By default this method calls fn n - 1 times, where n is the length of the array. On the first call it is passed the first and second elements in the array. The result of that callback will then be passed into the next iteration until it reaches the end, where the accumulated value will be returned as the final result. If init is passed, it will call fn one extra time in the beginning passing in init along with the first element. This method is only provided for those browsers that do not support it natively.
  • reduceRight(fn, init) Reduces the array to a single result by stepping through it from the right. By default this method calls fn n - 1 times, where n is the length of the array. On the first call it is passed the last and second to last elements in the array. The result of that callback will then be passed into the next iteration until it reaches the beginning, where the accumulated value will be returned as the final result. If init is passed, it will call fn one extra time in the beginning passing in init along with the last element. This method is only provided for those browsers that do not support it natively.
  • remove(f) Removes any element in the array that matches f. f will match a string, number, array, object, or alternately test against a function or regex. This method will change the array! Use exclude() for a non-destructive alias.
  • removeAtIndex(start, end) Removes element at start. If end is specified, removes the range between start and end. This method will change the array! If you don't intend the array to be changed use clone first.
  • shuffle() Alias for randomize().
  • some(fn, scope) Returns true if any element in the array matches f. scope is the this object. In addition to providing this method for browsers that don't support it natively, this enhanced method also directly accepts strings, numbers, deep objects, and arrays for f. any() and has are provided as aliases.
  • sortBy(property, desc = false) Sorts the array by property. property may be a function or a string acting as a shortcut. desc will sort the array in descending order.
  • split(f) Splits the array on any element that matches f. f will match a string, number, array, object, or alternately test against a function or regex.
  • subtract(a) Returns an array containing only the elements that do not exist in a.
  • sum(map) Sums all values in the array. map may be a function mapping the value to be summed or a string acting as a shortcut.
  • to(index) Returns a slice of the array up to index.
  • union(a) Returns an array containing all elements in both arrays with duplicates removed.
  • abs()
  • unique() Removes all duplicate elements in the array.

Components: Date Methods

  • Date.create(d) Alternate Date constructor which understands various formats. Accepts a multitude of text formats, a timestamp, or another date as d. If no argument is given, date is assumed to be now. Date.create additionally can accept enumerated parameters as with the standard date constructor.
  • addDays(num) Adds num days to the date. Similar handling also applies to addHours(), addMilliseconds(), addMinutes(), addMonths(), addSeconds(), addWeeks(), and addYears().
  • advance(param) Sets the date forward. param can accept multiple formats including a single number as a timestamp, an object, or enumerated parameters (as with the Date constructor).
  • beginningOfDay() Sets the date to the beginning of the day. Similar handling also applies to beginningOfMonth(), beginningOfWeek(), and beginningOfYear().
  • clone() Clones the date.
  • daysAgo() Alias for daysUntil(). Similar aliases exist, i.e. hoursAgo(), millisecondsAgo(), minutesAgo(), monthsAgo(), secondsAgo(), weeksAgo(), and yearsAgo().
  • daysFromNow() Alias for daysSince() Similar aliases exist, i.e. hoursFromNow(), millisecondsFromNow(), minutesFromNow(), monthsFromNow(), secondsFromNow(), weeksFromNow(), and yearsAgo().
  • daysInMonth() Returns the number of days in the date's month.
  • daysSince(d) Returns the number of days since d. Similar handling also applies to hoursSince(d), millisecondsSince(d), minutesSince(d), monthsSince(d), secondsSince(d), weeksSince(d), and yearsSince(d).
  • daysUntil(d) Returns the number of days until d. Similar handling also applies to hoursUntil(d), millisecondsUntil(d), minutesUntil(d), monthsUntil(d), secondsUntil(d), weeksUntil(d), and yearsUntil(d).
  • endOfDay() Sets the date to the end of the day. Similar handling also applies to endOfWeek(), endOfMonth(), and endOfYear().
  • format(format) Formats the date for the given format. format will accept a number of tokens as well as pre-determined formats. It can also be 'relative', which will construct a relative, unit-adjusted offset to the current time. A function may also be passed here to allow more localization and more granular control. relative() is provided as an alias.
  • getUTCOffset(iso) Returns a string representation of the offset from UTC time. If iso is true the offset will be in ISO8601 format.
  • getUTCWeek() Gets the date's week (of the year) according to universal time.
  • getUTCWeekday() Alias for getUTCDay().
  • getWeek() Gets the date's week (of the year).
  • getWeekday() Alias for getDay().
  • is(d, margin) Returns true if the date is d. margin is to allow extra margin of error (in ms). d will accept a date object, timestamp, or text format. If not specified, d is assumed to be now. is additionally understands more generalized expressions like month/weekday names, 'today', etc. Dates will be compared to the precision implied in d.
  • isAfter(d, margin) Returns true if the date is after the d. margin is to allow extra margin of error (in ms). d will accept a date object, timestamp, or text format. If not specified, d is assumed to be now.
  • isBefore(d, margin) Returns true if the date is before d. margin is to allow extra margin of error (in ms). d will accept a date object, timestamp, or text format. If not specified, d is assumed to be now.
  • isBetween(d1, d2, margin) Returns true if the date falls between d1 and d2. margin is to allow extra margin of error (in ms). d1 and d2 will accept a date object, timestamp, or text format. If not specified, they are assumed to be now.
  • isFuture() Returns true if the date is in the future. Similar boolean handling exists for isLastMonth(), isLastWeek(), isLastYear(), isLeapYear(), isNextMonth(), isNextWeek(), isNextYear(), isPast(), isThisMonth(), isThisWeek(), isThisYear(), isToday(), isTomorrow().
  • isUTC() Returns true if the date has no timezone offset.
  • isValid() Returns true if the date is valid.
  • iso() Alias for toISOString().
  • relative(fn) Shortcut for format('relative'). fn is a callback that can be used for more granular control over the resulting string. fn is passed 4 arguments: the adjusted value, adjusted unit, offset in milliseconds, and dir, which is -1 if the string is in the past and 1 if it is in the future.
  • resetTime() Resets the time in the date to 00:00:00.000.
  • rewind(param) Sets the date back. param can accept multiple formats including a single number as a timestamp, an object, or enumerated parameters (as with the Date constructor).
  • set(set, reset = false) Sets the date object. set can accept multiple formats including a single number as a timestamp, an object, or enumerated parameters (as with the Date constructor). If reset is set, the time will also be reset to 00:00:00.000.
  • setUTC(param) Sets the date object according to universal time. param can accept multiple formats including a single number as a timestamp, an object, or enumerated parameters (as with the Date constructor).
  • setUTCWeek(d) Sets the week (of the year) according to universal time. d must be an integer between 0 and 52. Similar handling exists for setWeek(d).
  • setUTCWeekday(d) Sets the weekday of the date according to universal time. d must be an integer between 0 and 6. Similar handling exists for setWeekday(d).
  • toISOString() Formats the string to ISO8601 format. This will always format as UTC time. Provided for browsers that do not support this method.

Example 1: Easily Parse Twitter JSONP Response

getLatestTweets(function(t) {
  var users = t.map('user').unique();
  var total = users.sum('statuses_count').format();
  var top   = users.max('followers_count').first();
  var count = top.followers_count.format();
  var since = Date.create(top.created_at);
  return 'Returned ' + users.length + ' Skookum team members with a total of ' + total + ' tweets.\n' +
         top.screen_name + ' is the most followed with ' + count + ' followers\n' +
         'and started tweeting ' + since.relative() + '.';
});
